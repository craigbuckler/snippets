owl - Optimalworks JavaScript Library

owl is offers modular components that do not use the global namespace, overwrite other variables, or apply prototype functions to core objects. The following components are provided.

  1. owl core

    Provides core functionality.

    file:
    owl.js
    requires:
    n/a

    1. owl.Version

      The owl version number.

    2. owl.UserAgent

      The browser user agent string.

    3. owl.Browser

      The browser name and version.

      owl.Browser.IE
      returns true if Internet Explorer
      owl.Browser.Mozilla
      returns true if Mozilla or Firefox
      owl.Browser.Opera
      returns true if Opera
      owl.Browser.Safari
      returns true if Safari
      owl.Browser.Version
      the browser version number as a string, e.g. 2.0.0.9
      owl.Browser.VerNum
      the browser version as a real number, e.g. 2

    4. owl.Number

      Basic number-handling functions.

      owl.Number.toInt(obj)
      pass any object to return an integer
      owl.Number.Sign(num)
      returns -1 for negative, 0 for zero, or 1 for positive numbers

    5. owl.String

      Basic string-handling functions.

      ow.String.Trim(str)
      returns a original string with whitespace removed from start and end
      ow.String.Clean(str)
      applies a reduced character set and reduces all whitespace to a single space
      ow.String.Pad(str, length, chr)
      returns str with a minimum length. The start of the string is padded with chr if required.
      ow.String.Format(str, params)
      replaces %0, %1, ... %n with values in the params array.

    6. owl.Array

      Basic array-handling functions.

      ow.Array.Is(obj)
      returns true if obj is an array
      owl.Array.Push(array, element)
      adds element to the end of the array, increasing its length
      owl.Array.Pop(array)
      returns the last element of the array, decreasing its length
      owl.Array.Make(array, default)
      will always return an array. The default is optional and only returned (as an array) if arr is falsy (undefined, false, null, etc.)

    7. owl.Each(obj, func)

      Iterates all items in obj (numbered i, from 0 to the length-1), and runs function func(obj[i], i). The iteration is stopped if func returns false.

    8. owl.Property

      Adds a property to an element or object using an owl namespace. Existing properties and methods should not conflict.

      owl.Property.Set(element, name, value)

      sets the 'name' property of an element to value, e.g.

      owl.Property.Set(document.getElementById("x"), "pName", 10);
      owl.Property.Get(element, name)
      returns the value for the 'name' property (or null)
      owl.Property.Exists(element, name)
      returns true if the name property is available
      owl.Property.Delete(element, name)
      deletes the property

    9. owl.Object

      Object manipulation and checking functions.

      owl.Object.Exists(object, item)
      returns true if item is a property or method of the object
      owl.Object.PropertyExists(object, item)
      returns true if item is a property of the object
      owl.Object.MethodExists(object, item)
      returns true if item is a method of the object
      owl.Object.Serialize(object)
      returns a JSON string representing all properties set within the object.
      owl.Object.DeSerialize(string)
      returns an object from a serialised string.

  2. owl.Http

    Returns server, page, and argument information.

    file:
    owl_http.js
    requires:
    n/a

    1. owl.Http.Arguments([url])

      returns an object containing all querystring values, e.g. if current URL is page.html?arg1=foo&arg2=bar

      var a = owl.Http.Arguments(); alert( a.arg1 ) // show "foo"

      Optionally, any page URL can be passed to the method.

    2. owl.Http.Argument(name)

      returns a single named querystring value (for the current URL only)

    4. owl.Cookie.Delete(name)

  4. owl.Image

    Provides image handling functions.

    file:
    owl_image.js
    requires:
    n/a

    1. owl.Image.Load(filename [, callback])

      loads an image. If defined, the callback function is called with the Image() object once the image download is complete

    2. owl.Image.IEpng(element, filename [, sizing])

  5. owl.Css

    Provides Cascading Style Sheet functionality.

    file:
    owl_css.js
    requires:
    n/a

    1. owl.Css.ClassExists(element(s), name)

      returns true if the element, or the array of elements, all have the class 'name' applied

    2. owl.Css.ClassApply(element(s), name)

      applies the class 'name' to all elements

    3. owl.Css.ClassRemove(element(s), name)

      removes the class 'name' from all elements

    4. owl.Css.Opacity(element(s), opacity [, autoHide])

      Sets the opacity of all elements to 'opacity', a value between 0 (fully transparent) to 100 (fully opaque).

      Visibility is set to visible or hidden as necessary, unless autoHide is set to false. This ensures cross-browser compatibility with browsers that do not support transparency.

      IE only supports transparency for elements with hasLayout.

    5. owl.Css.ComputedStyle(element, property)

      returns the computed style for an element, e.g.

      alert ( owl.Css.ComputedStyle ( owl.Dom.Get("h1") ), "margin-top"); // fetch H1 top margin

  6. owl.Dom

    Provides simple DOM access.

    file:
    owl_dom.js
    requires:
    owl.Css

    1. owl.Dom.ElementNode

      An element node type (1).

    2. owl.Dom.AttributeNode

      An attribute node type (2).

    3. owl.Dom.TextNode

      A text node type (3).

    4. owl.Dom.CommentNode

      A comment node type (8).

    5. owl.Dom.Get(csspath [, element(s)])

      Returns an array of nodes determined from a string containing one or more CSS-like selectors.

      selector description
      tag a standard tag, such as div, h1, p etc. Note that * refers to all tags.
      #id an element ID
      .class an element class name
      [attribute] an element with this attribute
      [attribute=value] an element that has this attribute set to value
      [attribute!=value] an element where the attribute is not set to the value (or the attribute is not defined)
      [attribute*=value] an element that has this attribute with this value somewhere in its definition, e.g. [attr*='foo'] matches 'foo' or 'fool'
      [attribute~=value] an element that has this attribute with this value within a space-separated definition, e.g. [attr~='foo'] matches 'abc foo' but not 'abc fool'

      Examples:

      // return all h1 and h2 nodes var d = owl.Dom.Get("h1, h2"); // returns nodes with a class of foo and a summary attribute of 'my table' var d = owl.Dom.Get(".foo[summary='my table']"); // return P nodes with a class of foo within the DIV node that has an ID of main var d = owl.Dom.Get("div#main p.foo"); // this does exactly the same var d = owl.Dom.Get("div#main"); var e = owl.Dom.Get("p.foo", d); // pass starting nodes

    6. owl.Dom.Descendents(element(s) [, level, condition])

      Returns and array of child nodes from the passed element(s) to an optional maximum level (1 = immediate children only) and a certain condition.

      Examples

      // get a starting node var d = owl.Dom.Get("#section"); // get all child nodes var c = owl.Dom.Descendents(d); // get immediate children only var c1 = owl.Dom.Descendents(d, 1); // get children up to level 2 that have at least one child of their own var c2 = owl.Dom.Descendents(d, 2, function(node) { return (node.childNodes.length > 1); });

    7. owl.Dom.Ancestors(element(s) [, nodename])

      Finds a node by name, e.g. div, ol, etc. for each element passed. If the passed node does not match, all parent nodes will be tested until a matching one is found. This can be useful when events return a child element of the node expected.

      Null is returned if no match is found. Passing a single node returns a single node, and passing an array (even with one element) returns an array.

      Examples

      // get all LI nodes var li = owl.Dom.Get("li"); // return single ancestors that match OL // UL lists will return null, unless that in turn is enclosed in an OL list var ol = owl.Dom.Ancestors(li, "ol");

    8. owl.Dom.Clone(fNode, tNode [, move, clear])

      Safely copies all child nodes from fNode to tNode.

      If move is set to true, then the child nodes are moved rather than copied. If it is set to false (or not specified), then the copied nodes (in tNode) will have conflicting IDs removed.

      If clear is set to true, tNodes children are removed prior to the copying process.

    9. owl.Dom.RemoveIDs(element(s))

      Removes any IDs defined for the element(s) children. This can be useful when nodes are cloned.

    10. owl.Dom.RemoveChildren(element(s))

      Removes all element(s) child nodes.

    11. owl.Dom.Text(element(s) [, string|array])

      If no second argument is defined, this method will return the text values for all nodes separated by carriage returns (\n).

      If the second argument is a string, every element will have that value assigned to its text node.

      If the second argument is an array of strings, the values will be assigned to the elements in order. If there are more elements than strings in the array, the last string will be repeated.

    Provides XML loading and manipulation functions.

    file:
    owl_xml.js

    1. owl.Xml.ElementNode

    2. owl.Xml.AttributeNode

    3. owl.Xml.TextNode

    4. owl.Xml.CommentNode

  8. owl.innerHTML

    Provides a safe innerHTML function that converts an HTML string and loads it into the DOM.

    file:
    owl_innerhtml.js
    requires:
    owl.Dom, owl.Xml

    1. owl.innerHTML(element(s), HTMLstring [, clear])

      A valid XHTML string must be passed. This will be converted to a DOM object and inserted into the element(s).

      Any existing element children are removed by default. Set clear to false to append the nodes instead.

      The final node added as a child of the last element is returned.

  9. owl.Event

    Provides cross-browser event handling.

    file:
    owl_event.js
    requires:
    n/a

    1. new owl.Event(element, type, handler[, priority])

      Define events for one or more elements and cleans up all events when the page is unloaded.

      new owl.Event(window, "load", MyStartFunction, 0); var e = new owl.Event(document.getElementsByTagName("p"), "click", MyEventHandler);
      element
      one or more DOM elements
      type
      the event type, e.g. load, unload, change, focus, blur, resize, submit, mouseover, mouseout, mousedown, mouseup, mousemove, click, keypress, etc.
      handler
      the function run when the event occurs (passes an owl.EventInformation object)
      priority
      a number indicating the precedence of the event - ones with a lower number are executed first (optional)

      The owl.Event object also supports the following methods:

      .Detach()
      cancels the event(s) so they cannot be fired again

    2. owl.EventInformation

      This object is passed to every event handler function as the first argument. The following properties and methods can be used:

      .Event
      the standard browser event object
      .Element
      returns the element the event was attached to, e.g. a table (read-only)
      .Target
      returnes the element that fired the event, e.g. a table cell (read-only)
      .Type
      the event type, e.g. click, mouseover, etc.
      .Key()

      returns an object with information about any key pressed during the event (see keyboard events):

      .Pressed
      the key that was pressed, e.g. 'a'
      .Function
      the function key that was pressed, e.g. tab, up, pageup, etc.
      .Shift
      true if Shift was pressed
      .Ctrl
      true if Ctrl was pressed
      .Alt
      true if Alt was pressed
      .Mouse()

      returns an object with information about the mouse location:

      .X
      the horizontal location of the cursor on the page
      .Y
      the vertical location of the cursor on the page
      .StopHandlers()
      cancels all further functions for this event that have a lower precedence
      .StopPropagation()
      cancels an event being fired for parent elements with the same event type
      .StopDefaultAction()
      stops the default action occurring, e.g. a form submit
      .Detach()
      cancels the event(s) so they cannot be fired again

      Keyboard events

      An element with a tabindex of zero becomes accessible from the keyboard:

      element.setAttribute("tabindex", 0); // set to -1 to disable keyboard control

      The element can now receive focus and blur events, then react to key presses.

      Note that keyboard handling is not consistent across browsers. The following codes are used in the table below:

      • stan: can return a standard key code
      • ascii: a real character has been returned, i.e. Shift+1 will return "!"
      • kbd: a keyboard code has been returned, e.g. Shift+1 will return "1". In most cases these will be similar to the ascii equivalent, but some punctuation keys can return strange codes.
      • ctrl: can return a control key code, such as up, down, pageup, etc.
      • cancel: the event can be cancelled at this point to ensure that the default action does not occur
        keydown keypress keyup
      fired when a key is pressed and before it is displayed just before the key character is displayed. This event is the only one that will return a true ASCII value for the key that has been pressed. just after the key character is displayed
      Firefox stan - kbd
      ctrl
      cancel      		 stan - ascii
      ctrl
      cancel      		 ctrl
      standard keys cannot be cancelled and may start a quick find
      Internet Explorer stan - kbd
      ctrl
      cancel      		 stan - ascii
      control keys are not captured at this point
      only standard key presses can be cancelled      		 stan - kbd
      ctrl
      Opera ctrl
      standard keys start browser shortcuts      		 stan - ascii
      ctrl
      cancel      		 ctrl
      standard keys start browser shortcuts

      To ensure cross-browser keyboard compatibility, define two keyboard events to handle the same action:

      • Define a keydown event. Modify the element then cancel the default action of a control key, e.g. pressing down does not scroll the page (ensures IE compatibility).
      • Create a keypress event. Modify the element then cancel the default action of a control key (ensures Opera compatibility. Note that Opera returns the same key codes for some standard and control keys, i.e. an apostrophe has the same code as the right cursor. Therefore, if you are checking for the right cursor key, check for the apostrophe too.)

  10. owl.Session

    Store and retrieve session values. This method is similar to using cookies, except that the values only live for the current browser session (they will survive if moving away from the domain and clicking back), no data is transferred to the server, and several megabytes of information can be stored.

    file:
    owl_session.js
    requires:
    owl.Event

    1. owl.Session.Store(name, value)

      Store a value or object by name, e.g.

      owl.Session.Store("var1", "my string"); owl.Session.Store("var2", { arg1: 1, arg2: "two" });

    2. owl.Session.Restore(name)

      Retrieves the session value/object for the stored name, or returns null if it does not exist.

    3. owl.Session.Clear()

      Resets all session variables.

  11. owl.Ajax

    Provides cross-browser remote web service calls. Implements Ajax using XMLHttpRequest (GET and POST) and <script> insertion. Any HTML form can be converted to use Ajax rather than standard web posts (using accessible Hijax techniques).

    file:
    owl_ajax.js
    requires:
    owl.Dom, owl.Xml

    1. new owl.Ajax([form, callback])

      Define a web service call object.

      form
      a DOM form node. If this is provided, Hijax will be implemented so Ajax is used when submit is clicked (if form validation events are implemented, an invalid form should run the owl.Event.StopHandler() method no ensure no posting occurs.
      callback
      the function run when the web service returns data, fails, times out, or is aborted. The callback function is passed the owl.Ajax object so the reponse can be examined. If a callback is passed without a form node, then the Ajax call will be made automatically.

      Example

      new owl.Ajax( owl.Dom.Get("form#myform"), function(ws) { alert(ws.Status.Complete+"\n"+ws.Response.Text); }); var ws = new owl.Ajax();

      The following properties and methods can be used:

      .ID
      an identifying name (set to the form ID if Hijax is used)
      .Form
      the form node (Hijax)
      .Hijax(form)
      implements Hijax functionality in the same way as using new owl.Ajax(form)
      .URL
      the web service URL
      .Method
      the web service method: GET (default), POST or SCRIPT
      .Async
      true for asynchronous web service calls (the default of true recommended). SCRIPT calls is always asynchronous.
      .Timeout
      the timeout in milliseconds (default 10000 = 10 seconds). Set to 0 to have no timeout
      .Callback

      defines functions that are run under certain conditions:

      .Start
      called just prior to making the web service request
      .Success
      called when a web service response is successful
      .Fail
      called when a web service response fails, e.g. a 404 error
      .Timeout
      called if the web service request exceeds the .Timeout
      .Abort
      called if the web service call is aborted
      .SetCallback(callback)
      sets the same callback function for all conditions (Start, Success, Fail, Timeout and Abort)
      .Argument(name, value)
      defines an argument that will be posted to the web service. If Hijax is used, all form values are automatically posted.
      .Call([callback])
      calls the web service. If a callback function is specified, that function will be used for all conditions.
      .Abort()
      aborts the web service call
      .HijaxEvent
      the event object used to hijax a form (can use .Detach() to remove hijax functionality)
      .Status

      returns the status of the web service call:

      .Active
      true if the request is still running
      .Complete
      true if the request has been completed
      .Success
      true if a web service response was returned
      .Timeout
      true if the web service request exceeds the .Timeout
      .Abort
      true if the web service call was aborted
      .Response

      returned data from the web service (both will be null if .Status.Success is false):

      .Text
      a text response
      .XML
      a returned XML document
      .Config

      configuration for all web services

      .AjaxIdentifier
      if set, this parameter is added as an argument to all web service calls. It's value is set to the number of Ajax calls made. This setting will prevent caching and allow Ajax calls to be identified (the default name is "ajax")

      Note about SCRIPT Methods

      Using a .Method of "SCRIPT" is inconsistent in Opera and Konqueror. Opera 8.5 and Konqueror always report a failure, whereas Opera 9.0 will never timeout or abort. To ensure cross-browser compatibility, a callback parameter could be passed to the script which runs that function once it has been loaded.

  12. owl.Timer

    Clock timer and calling functionality.

    file:
    owl_timer.js
    requires:
    n/a

    1. new owl.Timer([start, stop, step, pause, startDelay, stopDelay, callback])

      Defines a new timer:

      start
      the timer start value (default 0)
      stop
      the timer stop value (default 100)
      step
      the step used when moving from start to stop
      pause
      the delay between steps in milliseconds
      startDelay
      the time in milliseconds to wait before starting the timer
      stopDelay
      the time in milliseconds to wait after the timer has stopped
      callback

      the function that is run during every iteration of the timer. The function is passed the timer object, so it can analyse values or change settings.

      If the callback function is defined, the timer will start immediately.

      Example:

      // start a timer new owl.Timer(0, 100, 1, 20, 0, 0, function(t) { window.status = t.Value; });

      The following properties and methods can be used:

      .Value
      the current value of the timer, as it moves from StartValue to StopValue (read/write)
      .StartValue
      the starting value (read/write)
      .StopValue
      the stopping value (read/write)
      .Start()
      starts or restarts the timer. This must be done if no callback argument is provided when the timer is created.
      .Stop()
      stop the timer
      .Reverse()
      reverse the timer
      .SetStep(newstep)
      defines a new step value. The timer will start, stop (0 passed), or reverse as necessary.
      .GetStep()
      returns the current step value
      .CallBack
      the function called every timer iteration (read/write). The timer object is passed as the first argument.
      .OnStart
      the function called when the timer starts from .StartValue following any startDelay (read/write). The timer object is passed as the first argument.
      .OnStop
      the function called when the timer stops when reaching .StopValue following any stopDelay (read/write). The timer object is passed as the first argument.
      .OnReverse
      the function called if the timer is reversed (read/write). The timer object is passed as the first argument.

  13. owl.Screen

    Provides browser viewport and screen functionality.

    file:
    owl_screen.js
    requires:
    owl.Dom, owl.Timer

    1. owl.Screen.Location(element)

      Returns the .X and .Y page co-ordinates of a single element, e.g.

      var Loc = owl.Screen.Location(document.getElementById("x")); alert(Loc.X + " , " + Loc.Y);

    2. owl.Screen.ViewPort()

      Returns the .Width and .Height of the browser HTML window.

      alert( owl.Screen.ViewPort().Width );

    3. owl.Screen.ViewScroll()

      Returns the .X and .Y page scroll location.

      alert( owl.Screen.ViewScroll().Y );

    4. owl.Screen.Page()

      Returns the .Width and .Height of the whole page.

      alert( owl.Screen.Page().Height );

    5. owl.Screen.Resolution()

      Returns the OS desktop .Width, .Height, .AvailWidth (width minus taskbar and widgets), .AvailHeight, .AvailLeft, .AvailTop, and .ColorDepth.

      alert( "Available resolution: " + owl.Screen.Resolution().AvailWidth + " x " + owl.Screen.Resolution().AvailHeight );

    6. owl.Screen.ScrollTo(X, Y[, animate, callback])

      Scrolls the browser viewport to X, Y. The scroll is slowly animated if animate is not set to false. An optional callback function is run once the viewport is in position.

    7. owl.Screen.ScrollToElement(element, left, right, top, bottom[, absolute, animate, callback])

      Scrolls the browser viewport so that the element is contained within the left, right, top and bottom co-ordinates of the viewport. The co-ordinates are assumed to be percentages, but will be taken as pixel values if absolute is set to true.

      The scroll is slowly animated if animate is not set to false. An optional callback function is run once the viewport is in position.

      // ensure element is shown in the top 30% of the page owl.Screen.ScrollToElement(document.getElementById("x"), 0, 100, 0, 30);

  15. owl.Effect

    Provides dynamic animation and effects.

    file:
    owl_effect.js
    requires:
    owl.Css, owl.Event, owl.Timer

    1. owl.Effect.Config

      Sets defaults for all effects on the page. It provides the properties:

      .Timer.FadeHover
      the name of the timer property applied to elements using FadeHover
      .Timer.ShowHide
      the name of the timer property applied to elements using the generic Show and Hide functions
      .Timer.Fade
      the name of the timer property applied to elements being faded only
      .Timer.Height
      the name of the timer property applied to elements being collapsed or expanded
      .Frame.Start
      the starting frame (default 0)
      .Frame.End
      the ending frame (default 100)
      .Frame.Step
      the step when moving between .Frame.Start and .Frame.End (default 3)
      .Frame.Pause
      the number of milliseconds between frames (default 10)
      .FadeHover.Min
      the minimum opacity for a .FadeHover element
      .Height.ScrollVert
      true to open or close vertically
      .Height.ScrollHorz
      true to open or close horizontally

    2. owl.Effect.Type

      Provides a range of animation functions. Functions can be added, removed or overridden. Each function is passed a single element node and an owl.Timer object so its .Value can be used. Note that these functions can be passed in an array to .Show and .Hide.

      .Fade
      sets the opacity of the element to t.Value
      .Height
      sets the element height according to the child's height multiplied by t.Value/100. The height will therefore expand or contract.

    3. owl.Effect.FadeHover(element)

      fades out one or more elements, but fades them in on hover

    4. owl.Effect.Show(element, animationFunction [, callback])

      applies one or more effects to display the elements: a single animationFunction or an array of animationFunctions can be passed so multiple effects can be applied (e.g. fade in and expand the height). The optional callback function is called when the animation is complete.

    5. owl.Effect.Hide(element, animationFunction [, callback])

      itentical to .Show, except that the elements will be hidden

    6. owl.Effect.FadeIn(element [, callback])

      fades an element in from fully transparent to fully opaque. The optional callback function is called when the animation is complete.

    7. owl.Effect.FadeOut(element [, callback])

      identical to .FadeIn, except that the element fades out

    8. owl.Effect.Expand(element [, callback])

      expands an element to its full width and height. The optional callback function is called when the animation is complete.

    9. owl.Effect.Collapse(element [, callback])

      identical to .Expand, except that the element collapses

  16. owl.Overlay

    Provides page and element overlaying facilites.

    file:
    owl_overlay.js
    requires:
    owl.Css, owl.Dom, owl.Screen, owl.Timer

    1. owl.Overlay.PageFadeOut([callback, color, opacityMax, opacityStep, opacityPause])

      'fades' the page out using a chosen overlay colour, which has opacity changing from 0 to opacityMax in steps of opacityStep every opacityPause milliseconds. Once the fade out has been completed, the callback function is run.

      It is also possible to change the default parameters in owl.Overlay.Config:

      owl.Config.PageFadeColor = "#0000cc"; owl.Config.PageFadeMax = 60; owl.Config.PageFadeStep = 5; owl.Config.PageFadePause = 50;

    2. owl.Overlay.PageFadeIn([callback])

      reverses the fade out, returns the page to normal, and calls the optional callback function

    3. new owl.Overlay.Elements(topX, topY, bottomX, bottomY)

      Fixes elements that can not be overlaid in IE (select boxes, applets and iframes). In IE5.5 and IE6.0, IFRAMEs are used to overlay elements under the co-ordinates. In IE5.0, the elements are temporarily removed.

      The following properties and methods can be used:

      .Cover()
      hides/removes the elements (automatically called by the constructor)
      .Show()
      re-displays the elements

    4. owl.Overlay.CreateIframe()

      returns an empty IFRAME element which can be added to the DOM (used by the owl.Overlay.Elements)

  17. owl.Lightbox

    Creates an attractive image browser for any list of image thumbnails.

    file:
    owl_lightbox.js
    requires:
    owl.Image, owl.Css, owl.Dom, owl.Xml, owl.innerHTML, owl.Event, owl.Timer, owl.Screen, owl.Overlay, and lightbox styles and images.

    1. new owl.Lightbox(node)

      This creates a new lightbox for a set of images. The HTML for that set is formatted in a certain way, i.e. you have a parent node that contains one or more images that are contained in links to larger versions of that image. For example:

      <ol class="lightbox"> <-- lightbox is automatically started --> <li><a href="images/image1.jpg"> <img src="images/image1_thumb.jpg" width="120" height="160" alt="image 1" /> <strong>IMAGE 1</strong> <span>This is image 1 of 3</span> </a></li> <li><a href="images/image2.jpg"> <img src="images/image2_thumb.jpg" width="120" height="160" alt="image 2" /> <strong>IMAGE 2</strong> <span>This is image 2 of 3</span> </a></li> <li><a href="images/image3.jpg"> <img src="images/image3_thumb.jpg" width="120" height="160" alt="image 3" /> <strong>IMAGE 3</strong> <span>This is image 3 of 3</span> </a></li> </ol>

      If the parent node is assigned the class owl.Lightbox.Config.Container.Element and owl.Lightbox.Config.AutoStart is true, then the lightbox will start automatically and there is no need to create the object.

    2. owl.Lightbox.Config

      Sets defaults for all the lightboxes on the page. It provides the properties:

      .AutoStart
      automatically creates a lightbox for a node with a class set to owl.Lightbox.Config.Container.Element
      .Enabled
      if true, the lightbox effect will be enabled
      .ShowInfo
      if true, the animated information bar will appear at the bottom of the image
      .Magnify
      if true, images that are larger than the available area will be scrolled
      .Throttle
      the number of milliseconds to wait before the information bar or zooming occurs when the mouse moves over the image (this should not be too low, or mouse out/over events can clash)
      .Container.Element
      the class name of an element that will have the lightbox applied (default of "lightbox")
      .Container.ActiveClass
      the class name applied to a lightbox-enabled node. The lightbox uses this to ensure the effect cannot be applied to the same node more than once
      .Preload.All
      if true, all images will be preloaded and cached when the page is loaded. This provides a fast user experience, but requires more bandwidth
      .Preload.Next
      if true, the next image will be preloaded and cached. This still requires a little more bandwidth, but less than owl.Lightbox.Config.Preload.All. Note that owl.Lightbox.Config.Preload.All must be set to false for this to work
      .Size.Start
      the width and height of the lightbox window when it starts (in pixels)
      .Size.Minimum
      the minimum width/height of the lightbox window (pixels)
      .Size.WidthPad
      the width of the lightbox window will be the browser width minus this value (pixels)
      .Size.HeightPad
      the height of the lightbox window will be the browser height minus this value (pixels)
      .Animation.FramePause
      the number of milliseconds between frame animations
      .Animation.FadeStep
      the opacity/transparency step when animated between 0 and 100
      .Animation.SizeStep
      the number of pixels the lightbox window resizes during a frame
      .Animation.BarOpacity
      the opacity of the information bar (0 = fully transparent, 100 = fully opaque)
      .Animation.BarSteps
      the number of pixels the information bar moves during a frame
      .Animation.ZoomSteps
      the number of zooming frames when moving from fully zoomed out to zoomed in

    3. owl.Lightbox.CreateWindow

      Returns a node reference to the lightbox window.

    4. new owl.Lightbox.Image(linknode)

      An image-handling object used by the lightbox. Several properties and methods are available:

      .Node
      the linknode passed to the object
      .Src
      the URL of the image referenced in linknode
      .Pic
      the loaded image object
      .RealWidth
      the actual width of the image
      .RealHeight
      the actual height of the image
      .Ratio
      the current zoomed ratio of the image (0 to 1)
      .Width
      the current zoomed width (.RealWidth x .Ratio)
      .Height
      the current zoomed height (.RealHeight x .Ratio)
      .Loading
      if true, the image is being loaded
      .LoadCallback
      the function(s) to run once the image has loaded
      .Load(callback)
      load the image and run callback once complete
      .Resize(maxwidth, maxheight)
      calculates the .Ratio, .Width and .Height given maximum dimensions

  18. owl.Assess

    Changes a series of questions into a step-by-step assessment. Each answer can be assigned a score and the total is used to determine which result is shown.

    file:
    owl_assess.js
    requires:
    owl.Css, owl.Dom, owl.Xml, owl.innerHTML, owl.Event, owl.Timer, owl.Screen, owl.Effect, and assess styles and images.

    1. new owl.Assess(node)

      Creates a new assessment system for a set of questions that use XHTML such as:

      <div class="assess"> <!-- assessment is automatically started --> <!-- question list --> <ol> <li><span>Are you an animal?</span></li> <li><span>Are you grey?</span></li> <li><span>Do you have a trunk?</span></li> </ol> <!-- question element --> <p class="ask"></p> <!-- answer possibilities --> <ul> <li><a href="#1" class="tick" title="answer yes"><strong>yes</strong></a></li> <li><a href="#0" class="cross" title="answer no"><strong>no</strong></a></li> </ul> <!-- results --> <p>I don't know enough to work out what you are!</p> <p class="score2">Mmm, are you a grey animal?</p> <p class="score3">Are you an elephant?</p> </div>

      The <p class="ask"></p> and the <ul> can be positioned where necessary.

      The answer possibilities are all assigned a score in the link's href. The final score is used to determine which result sentence is shown (the first is the default).

      If the parent node is assigned the class owl.Assess.Config.Element and owl.Assess.Config.AutoStart is true, then the assessment will start automatically and there is no need to create the object.

    2. owl.Assess.Config

      Sets defaults for all the assessments on the page. It provides the properties:

      .AutoStart
      automatically creates an assessment for a node with a class set to owl.Assess.Config.Element
      .Element
      the class name of an element that will have the assessment applied (default of "assess")
      .QuestionListNodes
      the elements inside the assessment node that contain questions (default of "ul li")
      .QuestionNode
      the node containing the single asked question (default of "p.ask")
      .AnswerContainer
      the node containing a list of answer links (default of "ul)
      .ResultNodes
      the elements inside the assessment node that contain results (default of "p[class!='ask']")
      .QuestionText
      the question introduction text (default "<strong>Question [QN] of [QT]:</strong>"). The string [QT] is changed to the current question number, and [QT] is changed to the number of questions.
      .RestartHTML
      HTML containing a link that will restart the assessment (default "<p class="restart"><a href="#restart">Restart assessment...</a></p>"). This is dynamically added if the assessment is started successfully.
      .Effect.ShowQuestion
      the function that shows the question (default owl.Effect.FadeIn). It is always passed a node and possibly a callback function.
      .Effect.HideQuestion
      the function that hides the question (default owl.Effect.FadeOut). It is always passed a node and possibly a callback function.
      .Effect.ShowResult
      the function that shows the result sentence (default owl.Effect.FadeIn). It is always passed a node and possibly a callback function.
